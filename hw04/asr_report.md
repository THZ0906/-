# ASR 报告与选型理由（asr_report）

## 1. ASR 技术简介
ASR（Automatic Speech Recognition）即自动语音识别技术，目标是将语音信号转换为文本信息。ASR广泛应用于语音助手、语音输入、字幕生成、会议纪要、工业语音控制等场景。

本次实验调研并对比多种开源ASR方案，并在本地环境实现可运行的识别程序。

---

## 2. 开源 ASR 方案调研（至少三种）

### 2.1 OpenAI Whisper
Whisper 是 OpenAI 开源的端到端语音识别模型，基于 Transformer 结构训练，支持多语言识别，中文表现优秀。Whisper 可直接识别 wav/mp3/mp4 等多种格式文件，并且对噪声环境具有一定鲁棒性。

特点：
- 多语言支持强
- 中文识别效果优秀
- 模型较大，推理速度相对慢

---

### 2.2 Vosk
Vosk 是轻量级离线语音识别框架，支持多语言模型下载，能够在低算力设备上运行，并可用于实时识别场景。

特点：
- 轻量、离线部署方便
- 推理速度快
- 中文识别准确率通常低于 Whisper

---

### 2.3 FunASR（阿里开源）
FunASR 是阿里开源的语音识别工具包，中文识别能力较强，并支持流式识别、端到端模型、多任务语音处理等能力，适用于实时语音系统开发。

特点：
- 中文识别能力强
- 支持流式/实时识别
- 部署依赖较多，上手成本较高

---

## 3. 三种方案对比分析

| 对比维度 | Whisper | Vosk | FunASR |
|---------|---------|------|--------|
| 开源协议 | MIT | Apache 2.0 | Apache 2.0 |
| 中文识别效果 | 很好 | 一般 | 很好 |
| 模型体积 | 较大 | 小 | 中等/较大 |
| 推理速度 | 中等偏慢 | 快 | 中等 |
| 是否支持实时/流式 | 支持（需额外封装） | 支持 | 支持（原生支持较好） |
| 部署难度 | 中等（需torch+ffmpeg） | 简单 | 偏难（依赖多） |
| 是否支持离线 | 支持 | 支持 | 支持 |
| 适合场景 | 字幕生成、科研、工业录音分析 | 嵌入式、轻量语音控制 | 中文语音系统、实时语音服务 |

---

## 4. 选型理由
本次实验选择 Whisper 作为实现方案，原因如下：
1. Whisper 对中文语音识别准确率较高，适合实验展示。
2. Whisper 支持直接识别 MP4 音视频文件，使用方便。
3. Python API 简单，便于快速实现可运行程序。
4. 对噪声和语速变化鲁棒性较好。
